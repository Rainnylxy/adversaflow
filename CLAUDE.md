## 项目定位

AdversaFlow 是一个 AI 红队对抗测试平台，面向简历展示 + 面试讨论。核心目标是用"工程化中间件集成 + 业务复杂度"证明后台工程能力。

## 技术决策原则

- 业务代码用 Go (Gin)，AI Worker 用 Python (Celery)，不混用
- 中间件不做自研——用开源标准方案，重点是"为什么选它、怎么配、怎么保证可靠性"
- 每个模块都可以在面试中独立展开讨论
- Docker Compose 一键启动所有环境，方便演示

## 面试叙事优先级

1. 架构设计：为什么拆成这 6 个微服务？
2. 中间件选型：Kafka vs RabbitMQ 双消息队列的理由？
3. 异步链路：Gateway → Kafka → Celery → WebSocket 的完整数据流
4. 可靠性：消息不丢、Worker 不挂、超时兜底
5. 可观测：Prometheus 埋点 + Jaeger 链路追踪
